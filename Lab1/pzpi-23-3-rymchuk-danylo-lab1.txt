2 МЕТА РОБОТИ
     	Мета роботи: доопрацювати документ Vision and Scope програмної системи оренди спорядження для хобі та розваг, зокрема оновити розділ 3 (Scope and Limitations) для врахування всіх компонентів системи та планування інтеграції штучного інтелекту.
     У межах роботи необхідно:
* перенести назву та короткий опис системи з попередньої версії документа Vision and Scope;
* описати скоуп початкового випуску всієї системи, включаючи серверну частину, мобільний застосунок, Web-клієнт та IoT-компонент;
* продумати та описати інтеграцію штучного інтелекту у майбутніх релізах за трьома аспектами: застосування, збирання даних, навчання та використання;
* визначити неочевидні обмеження системи та функціонал, який навмисно виключений;
* оформити звіт відповідно до ДСТУ 3008:2015.
     
     	

	 




3 ОПИС ВИКОНАНОЇ РОБОТИ
	Назва системи: HobbyRent - програмна система оренди спорядження для хобі та розваг.
	Короткий опис системи: HobbyRent - це інтегрована платформа для оренди спеціалізованого спорядження, що автоматизує весь життєвий цикл оренди: від інтерактивного пошуку з урахуванням категорій та геолокації до моніторингу фізичного стану об'єктів у реальному часі за допомогою IoT-модулів. Система виступає як маркетплейс, що забезпечує взаємодію між власниками активів та орендарями, мінімізує операційні ризики завдяки системі рейтингів та застав, підвищує прозорість угод та забезпечує високу якість обслуговування для спільноти ентузіастів активного відпочинку.
	3.1 Оновлення розділу "Початковий випуск"
     У розділі 3.1 було розширено опис скоупу початкового випуску системи. Якщо попередня версія містила лише загальний опис реєстрації користувачів та базового каталогу, то оновлена версія детально описує всі чотири компоненти системи: серверну частину (Spring Boot + PostgreSQL), мобільний застосунок (Android/Kotlin), Web-застосунок (з адміністративною панеллю та інтернаціоналізацією) та IoT-клієнт (ESP32 + DHT22). Додано опис взаємодії між компонентами через REST API та єдину базу даних.
     3.2 Додавання розділу "Сфера дії наступних випусків"
     У розділі 3.2 було описано інтеграцію штучного інтелекту у майбутніх релізах системи за трьома ключовими аспектами:
     Застосування ШІ: визначено три напрямки використання - аналіз стану обладнання на основі IoT-даних для виявлення аномалій та прогнозування технічного обслуговування; рекомендаційна система для персоналізації пропозицій спорядження; динамічне ціноутворення для оптимізації вартості оренди.
     Збирання даних: описано чотири категорії даних для навчання моделей - телеметричні дані IoT (температура, вологість), дані про оренди (історія транзакцій), поведінкові дані користувачів (пошукові запити, перегляди) та метадані обладнання (категорії, рейтинги, стан).
     Навчання та використання моделей: визначено підходи до навчання - самостійне навчання моделі аналізу стану обладнання з використанням Isolation Forest та Autoencoder; використання готового API сервісу рекомендацій для початкового впровадження з подальшим переходом на власну модель; навчання моделі ціноутворення методами регресії з розгортанням як окремий мікросервіс.
     3.3 Розширення розділу "Обмеження та виключення"
     У розділі 3.3 було додано детальний опис неочевидних обмежень системи та навмисно виключеного функціоналу. Визначено п'ять неочевидних обмежень: обмеження одним типом IoT-датчика, відсутність буферизації даних при втраті мережі, відсутність офлайн-режиму, обмеження кількості одночасних IoT-пристроїв, логічна 	фіксація застав. Додано шість пунктів навмисно виключеного функціоналу з обґрунтуванням: онлайн-оплата, інтеграція з кур'єрськими службами, юридичний супровід угод, страхування майна, push-сповіщення, багатомовний контент.


4 ВИСНОВКИ
	У результаті виконання лабораторної роботи було доопрацьовано розділ 3 документа Vision and Scope програмної системи HobbyRent, що охоплює скоуп початкового випуску всієї системи (серверна частина, мобільний застосунок, Web-клієнт та IoT-компонент), планування інтеграції штучного інтелекту у майбутніх релізах за трьома аспектами (аналіз стану обладнання на основі IoT-даних, рекомендаційна система для орендарів, динамічне ціноутворення) та вичерпний перелік обмежень і навмисно виключеного функціоналу. Сформовано цілісне уявлення про архітектурну організацію системи, взаємодію її компонентів та перспективи розвитку з використанням сучасних технологій штучного інтелекту.
