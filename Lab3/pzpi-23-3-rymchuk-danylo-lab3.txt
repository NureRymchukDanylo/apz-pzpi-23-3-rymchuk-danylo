3 ОПИС ВИКОНАНОЇ РОБОТИ
	3.1 Загальні відомості та інженерні рішення
     Для розроблення front-end частини системи HobbyRent було обрано бібліотеку React 18 у поєднанні з утилітарним CSS-фреймворком Tailwind CSS. Такий вибір зумовлений необхідністю створення швидкого, адаптивного та компонентно-орієнтованого інтерфейсу без використання складних систем збірки, що дозволяє інтегрувати front-end безпосередньо в статичні ресурси Spring Boot сервера.
     Архітектура застосунку реалізована за принципом Single Page Application (SPA). Управління станом здійснюється за допомогою React Hooks (useState, useEffect), що дозволяє уникнути надмірної складності при одночасному збереженні реактивності інтерфейсу. Для обміну даними з серверною частиною (Spring Boot REST API) використовується вбудований Fetch API, який забезпечує асинхронні HTTP-запити (GET, POST, PUT, DELETE, PATCH) та обробку відповідей у форматі JSON.
     3.2 Інтернаціоналізація та локалізація
     Інтернаціоналізація (i18n) реалізована шляхом створення словникової структури (об'єкта translations), що містить ключі та відповідні текстові значення для української (uk) та англійської (en) мов. Перемикання мови здійснюється динамічно через зміну стану language без перезавантаження сторінки.
     Локалізація (l10n) забезпечує адаптацію даних під регіональні стандарти:
* Формат дати та часу: використовується метод toLocaleString() з передачею відповідної локалі (uk-UA або en-US), що гарантує правильний порядок елементів дати та розділювачів.
* Сортування текстових значень: застосовується метод localeCompare() з вказанням локалі, що забезпечує коректне алфавітне сортування з урахуванням національних особливостей (наприклад, української абетки).
     
     	3.3 Рольова модель та інтерфейси
     	Система підтримує розмежування інтерфейсів залежно від ролі користувача:
1. Інтерфейс звичайного користувача:
o Перегляд каталогу спорядження з фільтрацією та пошуком.
o Панель IoT-моніторингу для відстеження стану орендованого обладнання в реальному часі (заряд батареї, координати, сигнал).
o Оформлення оренди через модальне вікно з розрахунком вартості.
2. Інтерфейс адміністратора:
o Управління спорядженням (додавання, редагування, видалення).
o Управління користувачами (перегляд списку, зміна ролей через PATCH запит, видалення облікових записів).
o Управління даними: експорт каталогу у форматі JSON та CSV, імпорт даних з JSON-файлів, створення повних резервних копій (backup) системи з часовою міткою.

	3.4 Взаємодія компонентів web-застосунку
	Головний компонент App виступає точкою входу та центральним сховищем стану. Він містить стани для поточного користувача (user), обраного представлення (view), мови (language), списку спорядження (equipment) та списку користувачів (users).
     При зміні стану view (наприклад, при натисканні на кнопку "Контроль" в навігаційній панелі), головний компонент умовно рендерить відповідні підкомпоненти або блоки інтерфейсу (Каталог, Моніторинг, Адмін-панель).
     Взаємодія з іншими елементами системи відбувається виключно через REST API бекенду. Front-end відправляє запити на ендпоінти /api/equipment, /api/admin/users, /api/rentals. Бекенд обробляє запити, звертається до бази даних PostgreSQL та повертає структуровані JSON-дані, які фронтенд використовує для оновлення свого стану та перемалювання інтерфейсу. Для забезпечення безпеки та коректної роботи між доменами на стороні сервера налаштовано CORS (Cross-Origin Resource Sharing).

4 ВИСНОВКИ
	У результаті виконання лабораторної роботи було розроблено front-end частину програмної системи HobbyRent, яка повністю відповідає вимогам до сучасних веб-застосунків. Реалізовано розмежування інтерфейсів для звичайних користувачів та адміністраторів, що забезпечує зручне управління системою та даними. Впроваджено механізми інтернаціоналізації та локалізації, які дозволяють адаптувати інтерфейс під українську та англійську мови з урахуванням регіональних стандартів форматування даних. Адміністративна панель забезпечує повний цикл управління системою, включаючи модерацію користувачів, експорт та імпорт даних, а також створення резервних копій. Інтеграція з серверною частиною через REST API працює стабільно, що підтверджує коректність обраних архітектурних та інженерних рішень.
