3 ОПИС ВИКОНАНОЇ РОБОТИ
	3.1 Опис проекту
	HobbyRent — це інтегрована платформа для оренди спеціалізованого спорядження для хобі та розваг з IoT-моніторингом. Система вирішує проблему високої вартості спорядження для рідкісних видів активного відпочинку (туризм, риболовля, спорт), надаючи можливість оренди замість купівлі.
	Ключові особливості:
    • Маркетплейс для оренди спорядження між приватними особами
    • IoT-контроль стану спорядження в реальному часі
    • Автоматизований розрахунок вартості оренди
    • Адміністрування системи та управління користувачами
3.2 Архітектура системи
Система побудована за трирівневою архітектурою та складається з чотирьох основних компонентів:
    1. Backend (Spring Boot 3): REST API, бізнес-логіка, робота з базою даних
    2. Web-клієнт (React 18): інтерфейс для користувачів та адміністраторів
    3. Мобільний застосунок (Kotlin + Jetpack Compose): нативний Android-додаток
    4. IoT-клієнт (ESP32 + DHT22): фізичний пристрій для збору телеметрії
Всі компоненти взаємодіють через REST API, дані зберігаються в PostgreSQL.
3.3 Виконані функції
Backend:
    • CRUD операції для спорядження, користувачів, оренд
    • Бізнес-логіка розрахунку вартості (знижка 15% при оренді >24 год)
    • Адмін-панель: управління ролями, експорт/імпорт даних, backup
    • Прийом та зберігання IoT-телеметрії
    • Система авторизації

Web-клієнт:
    • Каталог спорядження з фільтрами (пошук, категорія, доступність)
    • Оформлення оренди з розрахунком вартості
    • IoT-моніторинг в реальному часі (батарея, GPS, температура)
    • Інтернаціоналізація (UA/EN)
    • Адмін-панель з повним функціоналом

Мобільний застосунок:
    • Нативний Android-додаток на Kotlin
    • Архітектура MVVM
    • Каталог, оренда, IoT-моніторинг
    • Інтеграція з backend через Retrofit

IoT-клієнт:
    • ESP32 + датчик DHT22 
    • Циклічна відправка даних кожні 10 секунд
    • Інтеграція з хмарними тунелями

Масштабування:
    • Docker Compose з 3 інстансами Spring Boot
    • Nginx Load Balancer
    • Навантажувальне тестування (Locust)
3.4 Заплановані, але не виконані функції
    • Повноцінна система авторизації (JWT + Spring Security)
    • Онлайн-оплата (Stripe, LiqPay)
    • Інтеграція з кур'єрськими службами
    • Розширена IoT-інтеграція (GPS-трекери, датчики удару)
    • Машинне навчання для рекомендацій
    • Push-сповіщення
    • Страхування майна
4 ВИСНОВКИ
У результаті виконання лабораторної роботи було підготовлено та проведено презентацію програмної системи HobbyRent. Система реалізує повний цикл оренди спорядження з інтеграцією IoT-моніторингу.
Досягнення:
    • Розроблено 4 компоненти системи (Backend, Web, Mobile, IoT)
    • Реалізовано 80% запланованого функціоналу
    • Проведено навантажувальне тестування
    • Система готова до демонстрації та подальшого розвитку
Навички, отримані в ході проекту:
    • Архітектура ПЗ (мікросервіси, REST API, MVVM)
    • Робота з базами даних та ORM (JPA/Hibernate)
    • Мобільна розробка (Kotlin, Jetpack Compose)
    • IoT-інтеграція (ESP32, датчики)
    • Масштабування та навантажувальне тестування (Docker, Nginx, Locust)
    • Управління проектом (Git, GitHub, CI/CD)
Проект демонструє можливість створення повноцінної програмної системи з використанням сучасних технологій та архітектурних підходів.
